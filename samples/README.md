# FreeLook Sample Corpus

This directory contains committed sample files for design review, manual Quick Look validation, and selected future tests.

## Goals

- Keep representative files easy to inspect by eye.
- Cover the first-wave renderer formats and a few important edge shapes.
- Avoid committing oversized or non-UTF-8 fixtures when those are better generated inside tests.

## Current samples

- `markdown_showcase.md` — Markdown and practical GFM review surface
- `valid.json` — prettified JSON baseline
- `compact.json` — valid single-line JSON for prettify verification
- `invalid.json` — invalid JSON fallback and warning state
- `valid.xml` — prettified XML baseline
- `compact.xml` — valid single-line XML for prettify verification
- `invalid.xml` — invalid XML fallback and warning state
- `Example.swift` — Swift syntax highlighting sample
- `Example.java` — Java syntax highlighting sample
- `example.kt` — Kotlin syntax highlighting sample
- `example.kts` — Kotlin script syntax highlighting sample
- `Example.cs` — C# syntax highlighting sample

## Non-committed edge cases

The following should usually be generated by tests instead of committed here:

- files near or above the 500 KB preview cap,
- non-UTF-8 encoding fixtures such as Latin-1,
- parser stress cases that are too noisy for normal visual review.
